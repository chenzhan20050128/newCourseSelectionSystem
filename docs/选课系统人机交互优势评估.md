# 选课系统人机交互优势评估

## 评估范围与依据
- 参考资料：《人机交互课件节选》中的可用性目标、EEC 模型、黄金规则及 Nielsen 启发式原则。
- 评估对象：`frontend` 目录下现有 Vue 3 + Element Plus 前端代码及其核心交互组件（`Login.vue`、`Register.vue`、`Home.vue`、`CourseSearch.vue`、`MyCourses.vue`、`SmartCourseSelection.vue`、`CourseProgressDrawer.vue` 等）。
- 关注重点：从可用性与用户体验双重视角，梳理系统在学习性、效率、安全性、反馈性和愉悦性等方面的亮点，突出对学生用户的正向价值。

## 总体优势概述
- **以学生为核心的交互闭环**：登录到智能选课的全流程高度一致，符合课程选课典型任务链路，降低执行隔阂与评估隔阂。
- **一致的品牌视觉与语义**：紫色系配色、南大校徽、统一的排版层级，让用户在不同页面中保持同一心智模型，强化信任感。
- **多层次反馈机制**：从按钮状态、骨架动画、倒计时、Toast 到模态提示，覆盖操作前、中、后的关键环节，显著提升系统状态可见度。
- **辅助决策能力强**：课表可视化、学分进度华夫饼、智能对话推荐共同构建“全景式”认知框架，帮助学生完成复杂的选课决策。

## 可用性目标维度亮点
| 可用性目标 | 系统实现亮点 | 支撑示例 |
| --- | --- | --- |
| 易学性 (Learnability) | 初始登录页采用线性流程 + 自动加载轮次；课程查询默认触发一次搜索，帮助学生理解信息架构 | `Login.vue` 自动选择进行中的轮次；`CourseSearch.vue` 在 `onMounted` 中主动执行 `handleSearch({})` |
| 易记性 (Memorability) | 关键操作集中于顶部 Tab 和悬浮挂件；交互控件（按钮、标签）在不同场景复用一致语言 | `Home.vue` 的“三段式”标签；`CourseProgressDrawer.vue`、`MyCourses.vue` 使用相同色彩语义 |
| 高效率 (Efficiency) | 搜索支持组合条件、自动补全、快捷筛选；课表冲突检测和课程推荐减少反复尝试 | `AutocompleteInput.vue` Debounce + 聚合接口；`MyCourses.vue` 通过 `generateCourseBlocks` 一次性解决冲突布局 |
| 少出错 (Low error) | 前端校验、禁用按钮、轮次必选、验证码刷新等预防措施完善 | 登录/注册的 `required`、`pattern`、`minlength`；选课前检查 `selectedBatchId` |
| 安全性 (Safety) | 退课操作集中在详情弹窗与列表操作中，搭配确认与进度提示，避免误操作 | `MyCourses.vue` 中 `handleDrop` 双确认并提供操作消息队列 |
| 主观满意度 (Satisfaction) | 动画、卡片、渐变配色营造轻量体验；智能助手与弹性布局贴合现代应用风格 | 登录页 `slideUp` 动画；悬浮挂件使用 `cubic-bezier` 弹性过渡 |

## Nielsen 启发式视角的优势
1. **系统状态可见度**：
   - 登录、注册按钮提供加载态文案；选课、退课操作在 `CourseCard` 上实时显示 `message-toast`。
   - `Home.vue` 中批次信息可展开查看剩余时间，并以倒计时形式持续更新，易于评估当前状态。
2. **系统与现实世界的匹配**：
   - 用“学分”“节次”“周次”等专业词汇，同时保留自然语言描述（如“距离开始”“剩余时间”）。
   - 课表按照中国高校常用的 1-11 节时间段呈现，符合学生既有心智模型。
3. **用户控制权与自由**：
   - 顶部 Tab 允许学生在“课程查询”“我的课程”“智能选课”间快速切换。
   - 所有弹窗（课表、推荐、进度）均支持点击遮罩关闭，保留随时退出的能力。
4. **一致性与标准化**：
   - 统一的按钮圆角、渐变色、提示语气，以及多处沿用 Element Plus 规范，使系统具备稳定风格。
   - 轮次选择、多处推荐弹窗都采用相似的卡片布局和标签系统。
5. **错误预防**：
   - 未选择轮次时阻止选课；验证码与学号格式校验提前过滤无效输入。
   - `CourseProgressDrawer` 在未满足前置条件时给出明确提示，避免学生盲目尝试。
6. **识别优于回忆**：
   - 自动补全、下拉枚举、课表颜色标签降低记忆负担。
   - `CourseProgressDrawer` 的华夫饼视觉直接映射已修/已选状态，无需记忆数字。
7. **灵活性与效率**：
   - 智能选课助手支持快捷 prompt，帮助不同熟练度的学生快速获得建议。
   - `CourseSearch` 既支持关键字输入，也支持细粒度筛选和教师、学院自动补全。
8. **美学与极简设计**：
   - 卡片式布局留有充足留白，信息分层清晰；颜色用于语义强调而非装饰堆砌。
9. **帮助用户识别、诊断并恢复错误**：
   - 操作异常集中以 Toast/对话框提示，并附带后续行动建议（如“请先选择选课轮次”）。
10. **帮助与文档**：
    - 智能选课助手可被视为内嵌式帮助，通过自然语言提供说明与建议。

## 用户体验亮点
- **主动式信息推送**：课程轮次倒计时、选课进度挂件、学分华夫饼等组件主动揭示关键状态，避免学生错过时间节点。
- **情感化细节**：欢迎语、图标、动画、色彩渐变和微交互提升愉悦感，与课件中强调的“超越可用性”目标相吻合。
- **决策支持矩阵**：课程搜索（横向广度）+ 课表可视化（纵向时间冲突）+ 学分策略（纵深规划）+ AI 对话（即时咨询），形成互补闭环。
- **跨设备友好性**：布局中充分考虑响应式（`@media` 规则），保证在不同屏幕上仍能保持结构清晰。

## 进一步可深化的正向方向（保持优势基础上的拓展）
- **多角色可用性**：继续沿用人物角色方法，为教师或教务扩展视图，维持现有一致性优势。
- **情境化引导**：结合已有的智能助手，加入“首登向导”或“任务剧本”，帮助新手更快完成首次选课。
- **可访问性增强**：在当前语义化与色彩区分基础上增加高对比度模式或键盘导航，进一步放大“普遍可用性”的优势。

## 结论
该选课系统在交互设计上充分体现了课件中强调的“以人为中心”原则：流程自然、一致性强、反馈及时、辅助决策丰富。系统在易学性、效率、容错、用户控制、情感体验等维度均表现突出，是人机交互理论与实践良性结合的范例。后续若在角色扩展与可访问性方面持续优化，可进一步巩固其整体用户体验优势。
